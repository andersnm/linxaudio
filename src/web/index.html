Designer

Online audio synth/effect DSP graph editor and tracker/sequencer. The audio engine and DSP modules are compiled from C to asm.js for maximum performance and portability. The projects can be compiled to VST DLLs for desktop DAWs using desktop tools.

About

Graph designer: Modular audio framework to create audio plugins without a compiler or full development environment.

Sequencer/tracker composer: Automate MIDI and value pins in the project's interface using tracker style patterns.

Connect DSP modules through compatible pins to create new and more complex DSP modules. Select from a library of basic, precompiled DSP modules, such as f.ex filter, delay, oscillator, envelope generator. A module operates on values, buffers and midi messages exposed through its input and output pins.

A module pin is defined by its direction - either output or input - and a datatype. Output pins can be connected to compatible input pins. The possible datatypes are:

  • float
  • integer
  • float buffer
  • midi

Some modules support subgraph processing in which parts of their processing can be defered to a separate user-defined subgraph. F.ex the delay module uses a subgraph to process the delay buffer, and the polycontainer module clones the subgraph for each voice to implement polyphonic oscillators.

Tested in latest Firefox and Chrome. Uses IndexedDB for local storage, WebAudio for sound input and output, canvas for drawing, XMLHttpRequest for sharing projects through myjson.com, and asm.js through Emscripten for performance and cross platform compatilility.

This is still a project under very much development and anything may break at any time!

linxgen

linxgen.exe is a command line tool to compile JSON and link object files into a shared library executable (DLL) usable by your preferred DAW host.

Directory structure:

  • linxgen.exe
  • SimpleDelay.json
  • lib/cl-x86/*.obj

Usage to create SimpleDelayVST.dll plugin for VST hosts: 

    linxgen SimpleDelay.json vst-win32

Usage to create SimpleDelayBuzz.dll plugin for Buzz (or Buze): 

    linxgen SimpleDelay.json buzz-win32

Linxgen looks for object files in directories resolved from the target search paths in the JSON relative to the program directory. F.ex both targets "buzz-win32" and "vst-win32" specify "lib/cl-x86" in the search path.

Licensing and notes on commercial use

linxgen.exe is made from code primarily licenced under the LGPL. This license applies only to the graph compiler and linker, and *not* the executable output files created by the tool.

However, most of the standard modules included with linxgen are released under the LGPL as well, so any executables using standard modules will still fall under the LGPL!

The core graph code and host interface implementations are licenced under the BSD-license. This permits advanced users to create and redistribute plugins under any license, provided *none* of the LGPL standard libraries and modules are used.

Be aware that JSON graph description files also automatically fall under the LGPL when using the standard libraries and modules. The LGPL does not permit distribution of executable binaries unless all the source code is also made available. The JSON is a vital part of the plugin source code.

The idea is to keep the development of the core tools and libraries open, but still allow private and commercial use by advanced users.
